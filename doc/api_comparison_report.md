## 二、 深度特性分析

### 1. 中文索引能力 (Chinese Indexing)
*   **Brave Search**: 拥有独立的全球爬虫系统。在处理中文资讯（如国内政策、知乎、CSDN、博客园）时，能搜到很多“非主流”但极具价值的小众页面。如果你关心信息的**完整性**，Brave 是不二之选。
*   **Tavily**: 它的逻辑是“搜索结果的聚合者”。它不一定自己去存所有中文网页，但它懂得如何从主流站点中提取出最干货的结论。如果你关心信息的**提炼程度**，Tavily 更合适。

### 2. 免费政策与门槛 (Pricing & Threshold)
*   **Brave (2026 新规)**: 
    *   取消了“三无”免费版。
    *   现在需要填入信用卡。虽然每月赠送的 $5 额度通常够个人用，但“绑卡”这一步对部分国内用户存在门槛。
*   **Tavily**: 
    *   目前仍保持“仅需邮箱”即可注册。
    *   免费额度刷新及时（每月 1 号），且 API 返回结果中自带整理好的内容，能变相节省你调用大模型总结网页的 Token。

### 3. AI 适应性 (AI-Readiness)
*   **Tavily** 是赢家。它的输出直接就是 Markdown 或纯文本块，甚至包含了引用标注。这对 AutoClaw 这种需要快速决策的 Agent 来说，能显著降低“处理解析错误”的概率。
*   **Brave** 近期推出了 **LLM Context API**，追赶速度很快。它能把长网页切成碎片（Chunks）回传，减少了长文本溢出的风险。

## 四、 如何在 AutoClaw 中配置

您可以参考以下步骤在命令行或配置文件中切换：

### 1. 命令行切换
```bash
# 切换到 Tavily
openclaw configure --section web --key TAVILY_API_KEY --value [您的Key]

# 切换到 Brave
openclaw configure --section web --key BRAVE_API_KEY --value [您的Key]
```

- `TAVILY_API_KEY`
